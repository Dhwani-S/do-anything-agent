Query B  |  Tokyo activities with weather  (multi-goal + memory carryover)

Query C  |  Mom's birthday  (durable memory across two runs)
─────────────────────────────────────────────────────────────
  Run 1:
    "My mom's birthday is 15 May 2026. Remember that and give me a

    "When is mom's birthday?"

Expected Run 1: reminder files created in sandbox, fact persisted in
state/memory.json with kind="fact" and date extracted.

web search. Iteration count: 2.

use asyncio.run as entry point, prefer TaskGroup, avoid blocking calls

constraint. Verify critic node appears in the DAG graph.

sandbox_executor node. Coder produces valid Python; SandboxExecutor runs

QUICK REFERENCE — commands to run each query
────────────────────────────────────────────────────────────────────────

CLI (no frontend):
  .venv\Scripts\python.exe src\flow.py "<query here>"
  .venv\Scripts\python.exe src\flow.py --resume <session-id> ""

Frontend:
  Terminal 1:  .venv\Scripts\python.exe src\api.py
  Terminal 2:  cd frontend && npm run dev
  Browser:     http://localhost:5173  (or 5176 if ports busy)

Reset memory between attempts (S6/S7):
  Remove-Item src\state\memory.json -ErrorAction SilentlyContinue

Reset sessions between attempts (S8):
  Remove-Item src\state\sessions\* -Recurse -ErrorAction SilentlyContinue

------------- Test Query
Compare London, Paris, Berlin, Tokyo, and New York as cities for hosting a three-day international AI workshop next month.

For each city, find its current population, typical weather for next month, approximate flight accessibility, and one suitable family-friendly cultural activity for attendees.

In parallel, research the five cities. Then rank them using a Python scoring model that weighs population size, weather comfort, flight accessibility, and activity quality. Run the code and show the computed scores.

After scoring, have a critic verify whether the ranking is justified by the gathered evidence and whether the Python scoring logic is reasonable. If the critic finds a problem, revise the scoring once and rerun it.

Finish with the final ranked list, the winning city, the code output, and a short explanation of why the winner is best.
